Reactive programming
Consists of operations applied on data streams
Focuses on WHAT (not how) needs to be done when something notable happens


RxJS combinational operators:
- forkJoin([stream1, stream2]).subscribe((x) => console.log(x));
  // the most similar to promiseAll, waiting for ALL streams to finish
     before emiting the last values from streams
- zip(stream1, stream2).subscribe((x) => console.log(x));
  // shows the array of values of ALL streams while ALL streams emiting
- combineLatest(stream1, stream2).subscribe((x) => console.log(x));
  // shows the array of letest values from ALL streams while ANY stream emits
- merge(stream1, stream2).subscribe((x) => console.log(x));
  // merges streams in ONE stream and emits on timeline when streams emit

2 directions of binding view <-> source:
* source (component.ts) -> view (component.html - template)
  <element [...] = "<variable-from-source>"></element>
* view (component.html - template) -> source (component.ts)
  <element (...) = "<function-from-source>"></element>

in parent container in component.html directive *ngIf="<object>"
   this prevents rendering error of properties of the object if it doesnt
   exist or if it is null (instead of making in constructor dummy object)
   in component.ts example:
   @Input movie: Movie | null = null;
   rate() {
    if (this.movie) {
      this.movie.score += 0.1;
    }
   }

Reducer:
- EntityState<Type> instead of for example saving only Movies[], it saves 
  list of Ids fast to search and also dictionary with whole objects
  Additional properties can be added...
  Adapter is used for setting initial state
- Example of reducer:
  export interface MovieState extends EntityState<Movie> {
    selectedMovieId: string;
  }
  const adapter = createEntityAdapter<Movie>();
  const initialState: MoviesState = adapter.getInitialState({
    selectedMovieId: ""
  });
  export const moviesReducer = createReducer(initialState,
    on(Actions/namespace/.changeRaiting, (state, {movieId, rating}) => {
      const targetMovie = state.entities[movieId];
      if(targetMovie)
        return adapter.setOne({...targetMovie, score: rating}, state);
      else return state;
    }),
    on(Actions.loadMoviesSuccess, (state, {movies}) => adapter.SetAll(movies, state)
    )
  )

Docker:

  > docker pull postgres
  > docker image ls
  > docker run --name postgres -e POSTGRES_PASSWORD=mysecret -e
    POSTGRES_USER=postgres -d -p 5432:5432 /external port:internal container port/
    postgres /<image_from_which_to_create_container>/
  > docker ps 
- Beekeeper to connect to db:
  user: postgres, password: mysecret, default database: postgres
Put existing project into docker conatiner:
- create Dockerfile
  example:
    FROM node:18-alpine3.14
    WORKDIR /usr/src/app   // working directorium
    COPY package*.json ./  // copy to working dir just list of packets for instalation
    RUN npm install
    COPY . .               // copy all from current dir to working dir in container
    RUN npm run build
    CMD ["npm", "run", "start:dev"]
- > docker build -t nest-container .
  > docker run -dp 4000:3000 nest-container
